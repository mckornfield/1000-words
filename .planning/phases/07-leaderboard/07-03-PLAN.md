## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LeaderboardEntry data → rendered HTML | Server-provided display_name rendered via JSX — React auto-escapes, no XSS risk |
| resolveCosmetic itemId → emoji lookup | itemId is a store catalog key; lookup is against local static array, not user-supplied URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-07 | Information Disclosure | LeaderboardPage display_name rendering | accept | display_name is user-chosen public field. JSX rendering auto-escapes. No raw HTML injection possible via dangerouslySetInnerHTML (not used). |
| T-07-08 | Denial of Service | Promise.all fetch on every mount | accept | Two RPC calls on mount. No retry loop. Loading state prevents duplicate renders. Failed fetch shows toast, does not retry. |
| T-07-09 | Tampering | navigate("/leaderboard") from DashboardPage | accept | navigate() only modifies window.history; parseRoute() validates known paths. Navigating to /leaderboard requires requiresAuth check (already enforced in App.tsx useEffect). |
</threat_model>
